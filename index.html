How it started

Built for privacy. Made for you.

Artemis exists because a useful assistant should understand the machine where the work happens. It should be local-first, app-first, and clear about what it can and cannot do.

No cloud lock-in. No unnecessary tracking. No decorative privacy language. The goal is a desktop command layer that keeps control with the person at the keyboard.

user@artemis:~

> why artemis?

Because privacy should be the ground floor, not a toggle.

Because your assistant should work for your machine, not someone else's dashboard.

Safety boundary

Local-first does not mean reckless.

Type now in the app shell; voice/listening remains an explicit future runtime.

No microphone capture, screen capture, model switching, settings writes, or action execution is added by this site.

Future computer control stays behind approval gates, permissions, traces, and audits.

Safe action system work stays roadmap-only until a later approval-gated sprint.

No tracking, analytics, remote scripts, remote images, external fonts, or paid API dependency.

Roadmap

From local assistant to safe desktop command center.

  1. Now: terminal UI, dashboard/status bridge, local voice path, memory/project continuity, and typed app-shell stub.
  2. Next: richer app shell surfaces for chat, safe activity, models, screen state, memory, and settings previews.
  3. Later: voice runtime, explicit permissions, local model routing, and the approval-gated computer-action layer.
  4. Future: installer, updater, downloadable desktop assistant, and safer automation.
